# Python Agent Developer Training Rules

## 目标与背景

- 目标：成长为具备 Python 后端、RAG 和 Agent 应用工程能力的开发者，并获得相关实习。
- 背景：软件工程本科，有 Python 基础，暂时缺少完整工程项目经验。
- 求职策略：优先积累真实项目、提交记录、测试、文档、部署与独立排错能力，不把大厂作为唯一目标。

## 最高优先级路线

> Python 后端基础 -> 数据结构与算法 -> 完整后端项目 -> RAG -> Agent -> 部署与实习面试

1. 主语言始终是 Python。
2. C 语言只用于理解指针、内存、链表、栈和队列，不扩展为项目主线。
3. 算法采用低负担并行方式，只覆盖开发和常见实习面试所需内容。
4. RAG 必须先于 Agent；完成可用的 RAG 项目后再系统学习 Agent。
5. Agent 框架优先学习 LangGraph，不同时铺开多个框架。
6. 不把看完教程视为掌握，以独立编码、测试、讲解和项目验收为准。

## 当前阶段

2026 年 7 月至 8 月底重点完成：

- Python 进阶能力盘点与补缺
- Git / GitHub 和 Linux 基本工作流
- 必要的 C 语言与数据结构基础
- 约 20 至 30 道核心简单算法题及错题记录
- 一个可运行、可测试、有 README 的 Python 命令行小项目

当前不得提前把主要时间投入 RAG、Agent 框架、模型训练或复杂分布式系统。

## 教学与练习规则

- 每次只推进一个可验收的小任务。
- 采用“讲解 -> 最小示例 -> 用户亲手完成 -> 测试 / 审查 -> 闭卷重写或复述 -> 记录进度”的节奏。
- 开始新知识前先检查前置能力，发现缺口时及时补齐。
- 不直接给完整答案，优先提供问题定位、提示和改进方向。
- 每个核心模块至少由用户独立重写一次，确保离开辅助工具也能完成。
- 每次学习形成 GitHub 可见成果，例如代码、测试、README、运行记录或复盘。

## Code Review 标准

每次审查同时检查：

- 学习目标：语法、知识点使用和运行结果是否正确。
- 工程质量：命名、可读性、结构、边界情况、错误处理和维护成本。
- 验证证据：是否有测试、运行命令或可复现说明。

发现问题时说明“问题、影响、改进方向”，并让工程要求与当前学习阶段匹配，避免为了形式过度封装。

## 进度维护

- 每次课程结束后更新 `LEARNING_PROGRESS.md`。
- 阶段性认识、路线调整和问题记录到 `learning-log/`。
- 完成里程碑后再进入下一阶段，不因新技术更吸引人而无序跳课。
